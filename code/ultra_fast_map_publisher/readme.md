内存映射（mmap）：不把整个 PCD 文件读入内存，而是映射到虚拟内存，适合 GB 级大地图；
自定义 Fast Format：剔除 PCD 冗余头信息，仅存储 xyz 浮点数据，加载速度比原生 PCD 快数倍；
多线程并行：拆分点云数据为批次，多线程解析，充分利用 CPU 多核；
缓存机制：PCD 转 Fast Format 后缓存，后续加载直接用 Fast Format；
进度可视化：大地图加载时能看到实时进度，避免 “卡死后不知道状态”；
统计信息发布：额外发布地图边界、加载时间、点数等元信息，方便上层应用使用。

pcl_ros/pcd_to_pointcloud：像用记事本打开 1000 页的 PDF，需要先把整个 PDF 加载到内存，再逐行读，内存不够就卡死；
自定义代码：像用 PDF 阅读器的 “按需加载” 功能，只加载当前看的几页（mmap），还能多人分工读（多线程），读完后还能把 PDF 转成更轻量的 TXT 格式（Fast Format），下次读更快。
